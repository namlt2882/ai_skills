# Trading Experience Knowledge Classification

## When to use this skill
Use this skill when you need to systematically organize, classify, and process trading knowledge in a structured way that allows AI agents to understand, connect, and verify different aspects of trading information. This skill is particularly useful for building comprehensive trading knowledge bases that can evolve and self-correct over time.

## When NOT to use this skill
Don't use this skill for basic trading queries that don't require structured knowledge classification, or when you simply need quick market analysis without building a persistent knowledge framework.

## Inputs required from the user
- Specific trading knowledge domain to classify (e.g., technical analysis, fundamental analysis, risk management, market psychology)
- Existing knowledge sources or materials to be classified
- Specific aspects or categories for knowledge organization

## Workflow

1. Identify the primary trading knowledge domain to be classified
2. Define the interconnected aspects that will form the knowledge structure
3. Organize existing knowledge into the defined aspects
4. Establish deterministic connections between related aspects
5. Implement verification mechanisms to identify outdated or incorrect knowledge
6. Create correction protocols for identified knowledge gaps or errors
7. Document the knowledge classification system for future reference

### Systematic Trading Decision Workflow

For rapid trading decisions, follow this integrated workflow:

1. **Market Analysis** → Identify trading signals and market conditions
2. **News Impact Assessment** → Evaluate upcoming/past news events and their impact
3. **Risk Management** → Calculate position size and set stop loss using evaluation frameworks
4. **Execution Methods** → Apply systematic checklist for opening trades
5. **Profit/Loss Decisions** → Use decision matrices for profit taking and loss management
6. **Trading Psychology** → Maintain discipline throughout the process
7. **Performance Metrics** → Track and analyze outcomes for continuous improvement

This workflow ensures all critical trading decisions are made systematically using documented frameworks rather than relying on intuition or emotion.

## Examples

### Example 1: Classifying technical analysis knowledge
- Input: Various TA indicators and strategies
- Process: Organize by timeframes, reliability, market conditions
- Output: Structured knowledge base with interconnections

### Example 2: Systematic trade execution workflow
- Input: Trading signal from market analysis
- Process:
  1. Validate entry signal using Execution Methods checklist
  2. Calculate position size using Risk Management frameworks
  3. Set stop loss using technical and volatility-based methods
  4. Define profit targets using Profit/Loss decision matrix
  5. Execute trade following Execution Methods best practices
- Output: Structured trade with documented risk parameters

### Example 3: News impact assessment integration
- Input: High-impact economic event (e.g., FOMC decision)
- Process:
  1. Classify news using News Impact Assessment framework
  2. Evaluate pre-news position management strategies
  3. Monitor real-time market reaction using impact assessment checklist
  4. Apply post-news trading strategies based on market response
  5. Adjust risk parameters using Risk Management stop loss evaluation
- Output: Systematic approach to trading around news events

### Example 4: Profit/Loss decision integration
- Input: Open position reaching profit target
- Process:
  1. Assess market condition using Profit/Loss decision matrix
  2. Evaluate profit taking strategy (single target, partial, trailing)
  3. Execute profit taking using Execution Methods
  4. Adjust stop loss for remaining position using Risk Management
  5. Document outcome for Performance Metrics analysis
- Output: Systematic profit management with documented decision process

## Troubleshooting

- If knowledge aspects seem disconnected: Review the linking mechanisms between aspects
- If knowledge becomes outdated quickly: Implement more frequent verification cycles
- If classification is too rigid: Add flexibility mechanisms for evolving knowledge
